# Course Requirements Summary

This summary keeps only the BUSA 649 course requirements that affect the repo and delivery plan.

## Course Frame

- Course: BUSA 649 Community Analytics Projects.
- Term: Summer 2026, running from June 1 to July 30.
- Project model: team-based community analytics work using public data.
- Expected scale: approximately 100 hours of work per student.
- Required output: a working community deliverable that a real person can use. A notebook or slide deck alone does not qualify.

## Required Deliverables

| Deliverable | Weight | Deadline |
| --- | ---: | --- |
| Project Proposal PDF, 3-4 pages | 15% | 2026-06-12 |
| Progress Report 1, group and individual forms | 5% total | 2026-06-21 |
| Progress Report 2, group and individual forms | 5% total | 2026-07-05 |
| Progress Report 3, group and individual forms | 5% total | 2026-07-19 |
| Progress Report 4, group and individual forms | 5% total | 2026-07-26 |

## Proposal Requirements That Still Govern The Project

The proposal and follow-up work must show:

- a real community problem and clearly bounded scope;
- evidence that data exists, was downloaded or sampled, and is rich enough for the method;
- SMART objectives with measurable success criteria;
- a concrete deliverable and audience;
- a week-by-week timeline or Gantt chart;
- top risks with likelihood, impact, and mitigation;
- clear writing that fits a professional project handoff.

## Progress Report Expectations

Each progress-report cycle should record:

- progress this cycle in two or three plain-English sentences;
- goals for the next cycle as bullets;
- task allocation by teammate;
- blockers, if any;
- individual hours logged with short, specific descriptions.

## Final Deliverable Expectations

The final deliverable should be functional, deployed or easily deployable, and designed for non-technical users. For this repo, that means the navigator should run from documented commands, use public source-grounded data, show official links, and avoid unsupported advice.

## Final Presentation Expectations

The final presentation is live on 2026-07-30. It should explain the problem, data, method, working deliverable, community value, limitations, and next steps. The team should prepare a live demo, backup screenshots or video, and Q&A notes.
