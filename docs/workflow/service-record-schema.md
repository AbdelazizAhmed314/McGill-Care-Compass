# RAG Artifact Schema

This document defines the active v1 data contract for McGill Care Compass. The
project no longer uses the previous static record files as the data layer. The
active retrieval layer is the Silver RAG corpus generated by
[`scripts/data/build_rag_corpus.py`](../../scripts/data/build_rag_corpus.py).

## Active Artifacts

| Artifact | Layer | Purpose |
| --- | --- | --- |
| [`data/source-inputs/rag_seed_urls.csv`](../../data/source-inputs/rag_seed_urls.csv) | Config | Official seed URLs, source ownership, taxonomy, source terms, and crawl limits. |
| [`data/source-inputs/questionnaire_metadata_map.yml`](../../data/source-inputs/questionnaire_metadata_map.yml) | Config | Stable questionnaire IDs and keyword rules shared by the UI and RAG pipeline. |
| [`data/bronze/raw/rag_pages/*.html.gz`](../../data/README.md) | Bronze | Raw fetched HTML snapshots. |
| [`data/silver/processed/rag_pages/*.txt`](../../data/README.md) | Silver | Clean page text after boilerplate removal. |
| [`data/silver/datasets/rag_pages.csv`](../../data/silver/datasets/rag_pages.csv) | Silver | Page manifest, source metadata, drift hashes, and run stamp. |
| [`data/silver/datasets/rag_links.csv`](../../data/silver/datasets/rag_links.csv) | Silver | Discovered-link graph, link priority, crawl decisions, and run stamp. |
| [`data/silver/datasets/rag_chunks.csv`](../../data/silver/datasets/rag_chunks.csv) | Silver | Header-aware retrieval chunks, metadata tags, provenance, and run stamp. |
| [`data/silver/rag/rag_metadata.sqlite`](../../data/README.md) | Silver | SQLite copy of pages, links, and chunks. |
| [`data/silver/vector_store/chroma/`](../../data/README.md) | Silver | Local Chroma vector index rebuilt from [`rag_chunks.csv`](../../data/silver/datasets/rag_chunks.csv). |
| [`data/silver/reports/rag_pipeline_report.md`](../../data/silver/reports/rag_pipeline_report.md) | Silver | Human-readable v1 pipeline report. |
| [`data/silver/reports/rag_corpus_quality_report.md`](../../data/silver/reports/rag_corpus_quality_report.md) | Silver | Chunk-quality warnings and cleaning metrics. |
| [`data/silver/reports/rag_run_manifest.json`](../../data/silver/reports/rag_run_manifest.json) | Silver | Machine-readable manifest with version, config hashes, artifact hashes, and counts. |

Gold has no dataset in v1. A Gold dataset requires explicit team review of
Silver outputs.

## Shared Run Fields

Every page, link, and chunk row must include the same governance fields:

| Field | Purpose |
| --- | --- |
| `pipeline_version` | Data pipeline version that generated the row. Current v1 value: `1.0.0`. |
| `pipeline_run_id` | UTC run identifier, for example `20260621T193540Z`. |
| `artifact_schema_version` | Version of the CSV/SQLite artifact schema. Current value: `1`. |
| `generated_at` | UTC timestamp for the pipeline run. |
| `questionnaire_metadata_version` | Version from [`questionnaire_metadata_map.yml`](../../data/source-inputs/questionnaire_metadata_map.yml). |
| `seed_config_hash` | SHA-256 hash of [`rag_seed_urls.csv`](../../data/source-inputs/rag_seed_urls.csv). |
| `questionnaire_config_hash` | SHA-256 hash of [`questionnaire_metadata_map.yml`](../../data/source-inputs/questionnaire_metadata_map.yml). |
| `crawl_config_hash` | SHA-256 hash of crawl settings and versioned algorithm settings. |
| `chunking_config_version` | Version of the chunking logic. |
| `link_priority_config_version` | Version of the link-priority logic. |
| `embedding_model` | Embedding model used to build vectors. |

The validator checks that the CSV rows, manifest, SQLite DB, and Chroma count
belong to the same generated corpus.

## Page Schema

[`rag_pages.csv`](../../data/silver/datasets/rag_pages.csv) records one crawled or attempted page per canonical URL.

| Field group | Fields |
| --- | --- |
| Identity | `canonical_url`, `url`, `url_hash`, `seed_url_id`, `source_url`, `seed_url`, `parent_url`, `crawl_depth` |
| Source | `domain`, `source_group`, `source_owner`, `source_publisher`, `authority_level`, `terms_url`, `licence_or_terms`, `crawl_notes` |
| Crawl limits | `allowed_to_crawl`, `max_depth`, `max_pages_from_seed` |
| Taxonomy | `category_id`, `category_label` |
| Fetch evidence | `page_title`, `http_status`, `final_url`, `content_type`, `retrieved_at`, `source_updated_at`, `fetch_error` |
| Ranking | `source_priority_rank`, `freshness_score` |
| Storage and drift | `raw_html_path`, `clean_text_path`, `raw_html_hash`, `clean_text_hash`, `links_hash`, `previous_clean_text_hash`, `drift_status` |
| Links | `outbound_links`, `in_scope_links` |
| User context defaults | `student_type`, `jurisdiction`, `language`, legacy `risk_level` topic-sensitivity metadata |
| Governance | all shared run fields |

`drift_status` values are `new`, `unchanged`, `changed`, `fetch_failed`, or
`skipped`.

## Link Schema

[`rag_links.csv`](../../data/silver/datasets/rag_links.csv) records every discovered link, including links that were not
crawled.

| Field | Purpose |
| --- | --- |
| `source_canonical_url` | Page where the link appeared. |
| `target_canonical_url` | Canonicalized link target. |
| `anchor_text` | Visible link text. |
| `source_section_heading` | Section heading near the link. |
| `link_priority_score` | Deterministic score used to queue better links first. |
| `link_priority_reasons` | Audit reasons such as questionnaire terms, service terms, category matches, or deprioritized terms. |
| `link_type` | `in_scope`, `external`, `mailto`, `tel`, `file`, or `skipped`. |
| `crawl_depth` | Depth the target would have if queued. |
| `crawl_decision` | `queued` or `not_crawled`. |
| `skip_reason` | Reason the link was not queued, such as `depth_limit`, `duplicate_url`, `outside_allowed_path`, or `per_page_link_limit`. |
| Shared run fields | Version and manifest linkage. |

## Chunk Schema

[`rag_chunks.csv`](../../data/silver/datasets/rag_chunks.csv) is the retrieval contract for the future app.

| Field group | Fields |
| --- | --- |
| Identity | `chunk_id`, `canonical_url`, `url`, `url_hash`, `seed_url_id`, `chunk_index` |
| Text | `section_heading`, `heading_path`, `chunk_text`, `embedding_text`, `token_count`, `nearby_links` |
| Taxonomy | `category_id`, `category_label` |
| Source | `domain`, `source_group`, `source_owner`, `source_publisher`, `authority_level`, `terms_url`, `licence_or_terms` |
| Time and ranking | `retrieved_at`, `source_updated_at`, `source_priority_rank`, `freshness_score` |
| Drift | `content_hash`, `section_hash` |
| Questionnaire tags | `info_type_tags`, `has_contact_info`, `has_required_docs`, `has_eligibility`, `has_costs_coverage`, `has_location`, `has_deadlines`, `has_booking_steps`, `has_emergency_info` |
| User filters | `student_type`, `jurisdiction`, `language`, legacy `risk_level` topic-sensitivity metadata |
| Review and label provenance | `review_status`, `label_method`, `label_confidence`, `vector_id` |
| Governance | all shared run fields |

`embedding_text` prepends the heading path to the body text so retrieval keeps
section context. `chunk_text` remains clean citation text.

## Retrieval Contract

The app should retrieve in two steps:

1. Apply deterministic filters from the questionnaire: `category_id`,
   `info_type_tags`, `student_type`, `jurisdiction`, `language`, and
   user-context fields. Treat `risk_level` as legacy topic-sensitivity metadata, not actual chunk-level danger.
2. Run semantic similarity search inside the filtered Chroma subset.

Returned chunks must expose:

- `chunk_text`
- `section_heading`
- `canonical_url`
- `nearby_links`
- `category_id`
- `info_type_tags`
- `retrieved_at`
- `source_updated_at`
- `review_status`
- `label_method`
- `label_confidence`
- `vector_id`
- legacy `risk_level` when still present
- source terms fields

The response layer can use those chunks to produce grounded next steps, contacts,
required documents, costs, and source citations. It must not make medical,
legal, immigration, tax, insurance, financial, or eligibility decisions.

## Source Authority

Retrieval reranking prefers source groups in this order when chunks are relevant
to the same need:

1. Canada
2. Quebec
3. Official healthcare systems
4. McGill
5. Other approved sources

Within the same source tier, fresher source-updated dates rank higher.

## Validation

Run:

```bash
uv run python scripts/data/validate_rag_corpus.py
uv run pytest -q
uv run ruff check .
```

The validator checks schema columns, source terms, chunk size, heading prefixes,
metadata tags, review/provenance fields, SQLite parity, manifest consistency, artifact hashes, quality report presence, and Chroma vector count.
